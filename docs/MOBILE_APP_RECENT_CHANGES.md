# Recent Web Changes to Implement in Mobile App

This document lists recent features and fixes implemented on the **web client** that should be implemented in the **mobile app** for parity.

---

## 1. Payslip feature (new)

The web app has a full payslip flow. The mobile app currently has **no payslip screens**; these need to be added.

### Backend API (already exists – use same base URL prefix)

- **Base path:** `GET/POST /api/v1/payslip` (ensure mobile uses same API base; e.g. `getApiBaseUrl()` + `payslip/...`).

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST   | `/payslip/create` | Admin | Create payslip. Body: `employeeId`, `employeeName`, `employeeIdNo`, `designation`, `dateOfJoining`, `payPeriod`, `payDate`, `paidDays`, `lopDays`, `earnings` (basic, petrolAllowance, bikeAllowance, byBenefit, foodAllowance, incentives), `deductions` (taxPayable), `ratings` (timing, leave, workFb, incentive, firmFb). |
| GET    | `/payslip/all`   | Admin | List all payslips (for admin). |
| GET    | `/payslip/my`    | User  | List payslips for logged-in user’s employee record. |
| GET    | `/payslip/:id`   | User/Admin | Get one payslip (admin: any; user: own only). |

- Responses include **normalized** `earnings` and `deductions` (all keys present, numbers). Each payslip has `grossEarnings`, `totalDeductions`, `netPay` in the response.

### Admin – Payslip (Other Settings)

- **Payslip list screen**  
  - Fetch `GET /payslip/all`.  
  - Show list (e.g. pay period, employee name, net pay).  
  - Actions: open “View” and “Add Payslip”.

- **Add Payslip screen**  
  - Dropdown to select **employee** (from `GET /employee/all` or existing employee list).  
  - On select: auto-fill **Employee ID No** (e.g. `idCradNo` or `employeeIdNo`), **designation**, **date of joining** from employee.  
  - **Pay period:** date picker (store/send as "MMM YYYY", e.g. "Mar 2026").  
  - **Pay date**, **Paid days**, **LOP days**.  
  - **Earnings:** Basic, Petrol Allowance, Bike Allowance, By Benefit, Food Allowance, Incentives (numeric inputs).  
  - **Deductions:** Tax Payable.  
  - **Ratings:** Timing, Leave, Work FB, Incentive, Firm FB (e.g. 0–5 each).  
  - Submit as `POST /payslip/create` with the payload shape above.

- **Payslip view screen**  
  - Fetch `GET /payslip/:id`.  
  - Show: logo, “PAYSLIP”, “EMPLOYEE PAY SUMMARY”.  
  - Left: Employee Name, Employee ID, Designation, Date of Joining, Pay Period, Pay Date.  
  - Right: **Employee Net Pay** box (start/left-aligned): title, net pay amount, Paid Days \| LOP Days, then **five rating groups** (stars + labels: Timing, Leave, Work FB, Incentive, Firm FB) in a row.  
  - Table: **EARNINGS** (Basic, Petrol Allowance, Bike Allowance, By Benefit, Food Allowance, Incentives) with AMOUNT and YTD columns; **DEDUCTION** (Tax Payable, Total Deductions) with AMOUNT and YTD.  
  - **NET PAY** summary: Gross Earnings, Total Deductions.  
  - **TOTAL NET PAYABLE** with amount and “Total Net Payable ₹X (Indian Rupee X Only)”.  
  - Footer: “This document is system generated by Corp Culture; therefore, a signature is not required.”  
  - **Print:** On web we “print only payslip” (clone + hide app). On mobile you can use share/print or a PDF library to generate a single-page A4 payslip with the same layout and data.

### User – My Payslips

- **My Payslips screen**  
  - Fetch `GET /payslip/my`.  
  - List payslips (e.g. pay period, net pay).  
  - Tap to open **Payslip view** (same layout as above, read-only).  
  - Option to **print/share** (e.g. generate PDF or use system share/print).

### Data and display notes

- Use **fallbacks** so fields never appear blank: e.g. `employeeName ?? employeeId?.name ?? "-"`, `employeeIdNo ?? ""`, and same for designation, pay period; format missing numbers as 0 and dates as "-" when null/invalid.
- **Formatting:** Dates: `toLocaleDateString("en-IN")`; money: `₹${number.toLocaleString("en-IN")}`; handle null/NaN as "₹0".
- **Amount in words:** Backend does not provide this; web uses a `numberToWords` helper (e.g. for “Fifteen Thousand Only”). You can port that helper to the mobile app or implement a simple number-to-words function for the “Indian Rupee … Only” line.

## 2. Rental Enquiries – Pagination (web)

- **Web:** Admin Rental Enquiries list now has **client-side pagination**.  
  - State: `page` (1-based), `rowsPerPage` (default 10).  
  - Options for rows per page: 5, 10, 25, 50.  
  - UI: “Rows per page” dropdown, “X–Y of Z” label, and a pagination control (first/prev/page numbers/next/last).  
  - Page resets to 1 when **tab**, **rental type**, **rental title**, or **search** changes.  
  - List is filtered first (tab + filters + search), then sliced for current page.

**Mobile implementation options:**

- **Option A:** Add the same UX: show only `rowsPerPage` items per “page” and add a simple pager (e.g. “Previous” / “Next” and “Page 1 of N”) plus optional “Rows per page” picker.  
- **Option B:** Keep `FlatList` and add a “Load more” (or increase page size) when user scrolls to the end, or add a “Show 10 / 25 / 50” selector that re-slices the same filtered list.  
- **Option C:** If the API later supports server-side pagination (e.g. `?page=1&limit=10`), call it from mobile and map the same “page” and “rows per page” concepts to API params.

## 4. Optional: PDF / print on mobile

- Web no longer uses “Download PDF” from the view page; it uses **browser Print** (clone + print-only CSS).  
- On mobile, implement **print/share** via:  
  - System share sheet (e.g. share as PDF if the OS supports it), or
